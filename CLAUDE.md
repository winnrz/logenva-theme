# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Shopify prototype theme for **Longeva Pharmacy**, a pharmacy-led aesthetics business that supplies prescription-only medicines (POM) and practitioner products to registered healthcare practitioners. Base is unmodified **Shopify Dawn v15.4.1** (`config/settings_schema.json` → `theme_info`); brand work layers on top.

Goal: a working prototype that demonstrates judgement, not a production build. Brand direction is *premium, clinical, pharmacy-led, trustworthy* — not beauty-salon or generic ecommerce.

## Commands

Shopify CLI is installed globally (`shopify`). Common workflow:

- `shopify theme dev` — local dev server with hot-reload against the linked dev store
- `shopify theme push --unpublished` — push as a hidden theme on the store
- `shopify theme pull` — pull theme settings/assets from the store (used after merchants edit in admin)
- `shopify theme check` — Liquid/theme linter; run before committing template work

No Node `package.json`, no build step — assets in `assets/` are served as-is.

## Architecture

Standard Shopify theme layout. Conventions worth knowing before editing:

- **`layout/theme.liquid`** is the shell for every page; `password.liquid` is the password-page shell. All `<script>`/`<link>` includes live here, all loaded with `defer`.
- **`templates/*.json`** are JSON templates — they reference *sections* by name and pass settings. Liquid templates exist only for `gift_card.liquid`. To change a page's structure, edit the JSON; to change a section's markup, edit the section.
- **`sections/`** — page-composable blocks (homepage modules, `main-product`, `main-collection-product-grid`, etc.). `header-group.json` and `footer-group.json` define the persistent header/footer section groups.
- **`snippets/`** — reusable partials rendered with `{% render %}` (e.g. `card-product`, `price`, `buy-buttons`, `facets`). Prefer extending snippets over duplicating markup into sections.
- **`assets/`** — all JS and CSS, flat. Naming: `component-*.css` per UI component, `section-*.css` per section, top-level `*.js` per behaviour module (`cart.js`, `cart-drawer.js`, `facets.js`, `global.js` for shared utilities, `pubsub.js` for the event bus, `constants.js` for shared constants). No bundler — files are referenced by name via `{{ '<file>' | asset_url }}`.
- **`config/settings_schema.json`** defines theme editor settings (colors, typography, etc.); **`settings_data.json`** is merchant-edited state — treat as data, do not hand-edit unless seeding.
- **`locales/`** — translation strings. `en.default.json` is canonical; the file warns it is auto-generated by the Shopify admin language editor and may be overwritten.
- **`.shopify/metafields.json`** declares the product metafields the theme depends on (`shopify.medicine-drug-form`, `shopify.prescription-status` — both `list.metaobject_reference`).

## Build scope (from brief)

Nine functional areas: homepage, collection page, product pages, practitioner registration, gated product access, approval workflow, basket & checkout, VAT logic, mobile responsiveness. Catalogue is exactly three products:

| Product | Class | Public visibility | Pricing | Basket |
|---|---|---|---|---|
| Botox 50u | POM | hidden from public/pending (collections + nav) | approved only | approved only |
| Dermal Filler | Practitioner | visible to all | approved only | approved only |
| Skin Booster | Practitioner | visible to all | approved only | approved only |

User states: **public**, **pending** (post-registration, pre-approval), **approved practitioner**.

## Gating, approval, VAT — implementation guidance

Prefer **native Shopify primitives** over custom backend: customer tags, metafields, line item properties, cart attributes, Liquid conditionals. The brief explicitly says custom apps/backends are not expected.

- **Approval state** → Shopify customer tags (e.g. `pending`, `approved`). Gate price/buy-buttons/POM templates on `customer.tags contains 'approved'`. Admin approval = adding the tag (mock admin UI or documented manual flow is acceptable).
- **Practitioner products visibility** vs **POM hidden** — POM (`Botox 50u`) must not appear in collections, navigation, search, or be publicly indexed (regulatory). Use a tag or metafield on the product and filter in `main-collection-product-grid`, `predictive-search`, and add `noindex` in `meta-tags` when the visitor is not approved.
- **Practitioner products without approval** — show product page but replace price + buy-buttons block with a "Register as Practitioner" / "Request Access" CTA.
- **Prescription-linked vs clinic-stock pathway** — selected when an approved practitioner adds a practitioner product to basket. Acceptable mechanisms: product variants, line item properties, cart attributes. Selection drives VAT.
- **VAT logic** — per line item, conditional on supply pathway. Required scenarios (basket must demonstrate all four):
  1. POM only → 0% (zero-rated)
  2. POM + practitioner product on Rx → 0% / 0%
  3. POM + Rx practitioner + clinic-stock practitioner → 0% / 0% / 20%
  4. Practitioner product clinic stock only → 20%

  Doesn't need live tax integration — Liquid math over tags/metafields/line-item-properties is fine, but it must visibly compute and display per-line and totals in the cart/checkout summary.

- **SEO** — clean URLs (`/collections/injectables`, `/products/dermal-filler`), one H1 per page, sensible meta titles. **POM product pages must not be publicly indexed.**

## Working notes

- Don't hand-edit `config/settings_data.json` for things a merchant would set in the theme editor — push schema changes and let the editor populate.
- `locales/en.default.json` may be overwritten by Shopify admin; coordinate before adding strings, or accept they may need re-applying.
- Mobile responsiveness is non-negotiable per brief — test mobile width for nav, registration form, product cards, and CTA visibility.
- Reference site for ecommerce structure & practitioner gating UX: Two Face Aesthetics UK (the prototype should improve on it with a cleaner, more clinical feel).
